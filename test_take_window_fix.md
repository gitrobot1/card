# 选牌窗口修复测试指南

## 修复内容

### 1. 界徐盛破军技能
- **问题**：使用杀指定目标后，没有弹出选牌窗口
- **修复**：在 `playSha` 函数中，初始化破军后立即打开选牌窗口
- **预期行为**：
  1. 界徐盛使用杀指定目标
  2. 立即弹出选牌窗口，显示目标的所有牌（手牌、装备、判定区）
  3. 可以选择至多目标体力值张数的牌，置于目标的「营」区
  4. 选择完成后，目标需要出闪响应杀

### 2. 过河拆桥
- **问题**：使用过河拆桥后，直接拆掉第一张牌，没有选牌窗口
- **修复**：修改 `resolveGuoHe` 函数，打开选牌窗口
- **预期行为**：
  1. 使用过河拆桥指定目标
  2. 无懈可击结算完成后，弹出选牌窗口
  3. 可以选择目标的一张牌拆掉（放入弃牌堆）
  4. 可以选择「取消」跳过

### 3. 顺手牵羊
- **问题**：使用顺手牵羊后，直接获得第一张牌，没有选牌窗口
- **修复**：修改 `resolveTanNang` 函数，打开选牌窗口
- **预期行为**：
  1. 使用顺手牵羊指定目标
  2. 无懈可击结算完成后，弹出选牌窗口
  3. 可以选择目标的一张牌获得（放入自己的手牌）
  4. 可以选择「取消」跳过

## 注意事项

1. **前端需要刷新**：修改前端代码后，需要重新构建或刷新浏览器
2. **后端需要重启**：修改后端代码后，需要重新编译并重启服务器
3. **AI 自动选牌**：如果目标玩家是 AI，会自动选择第一张可用的牌
4. **取消选牌**：所有选牌窗口都支持「取消」操作，会跳过对应的效果

## 可能的问题

1. **选牌窗口不弹出**：
   - 检查后端是否正确设置了 `ResponseMode`
   - 检查前端 `handlers.ts` 中是否正确注册了 handler
   - 查看浏览器控制台是否有错误

2. **选牌后没有效果**：
   - 检查后端 `TakeOne` 函数是否正确执行
   - 检查牌是否正确移动到了目标区域
   - 查看后端日志是否有错误

3. **前端显示错误**：
   - 检查前端是否正确显示了目标的牌
   - 检查选牌窗口的标题和提示是否正确
   - 查看浏览器控制台是否有错误
